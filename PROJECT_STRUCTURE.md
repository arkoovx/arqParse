# 📦 Структура проекта ArcParse

## Основные модули (в порядке важности)

### **main.py** (195 строк)
Главная точка входа приложения. Содержит:
- Функция `main()` - главный workflow
- Этап 1: Скачивание конфигов
- Этап 2: Настройка прокси
- Этап 3: Тестирование конфигов
- Правильная обработка Ctrl+C
- Вывод итоговой сводки

**Улучшения v2.0:**
- ✅ Удалены неиспользуемые импорты
- ✅ Добавлена обработка прерываний (signal module)
- ✅ Добавлены подсказки для пользователя

---

### **xray_tester_simple.py** (460 строк)
Основной тестер конфигов. Использует Xray-core для тестирования VPN конфигов.

**Основные функции:**
- `test_batch()` - тестирует батч конфигов параллельно
- `_test_single_config()` - тестирует один конфиг
- `_create_xray_config()` - создает Xray конфиг из URL
- `_parse_vless_url()`, `_parse_vmess_url()` и т.д. - парсеры протоколов
- `_wait_for_port()` - ожидает открытия SOCKS порта
- `_cleanup_all()` - очистка процессов

**Улучшения v2.0:**
- ✅ Увеличена конкурентность: 100 → 150 потоков
- ✅ Сокращены таймауты для ускорения
- ✅ Упрощена HTTP retry стратегия
- ✅ Исправлен вывод прогресса (убран из потока)
- ✅ Улучшена обработка процессов и сигналов

---

### **testers.py** (56 строк)
Обертка над xray_tester_simple для унификации интерфейса.

**Функции:**
- `test_xray_configs()` - высокоуровневое API для тестирования Xray конфигов

**Улучшения v2.0:**
- ✅ Обновлены параметры по умолчанию (concurrency, timeout)

---

### **testers_mtproto.py** (100 строк)
Тестер MTProto прокси через TCP соединение.

**Функции:**
- `test_mtproto_configs()` - тестирует MTProto прокси параллельно
- `_test_single_mtproto()` - тестирует один MTProto

**Улучшения v2.0:**
- ✅ Добавлена проверка stop_flag в цикле
- ✅ Улучшена синхронизация между потоками

---

### **parser.py** (360 → 85 строк)
Парсеры конфигов и MTProto URL.

**Функции:**
- `parse_mtproto_url()` - парсит MTProto ссылки
- `read_configs_from_file()` - читает конфиги из файла
- `read_mtproto_from_file()` - читает MTProto из файла

**Улучшения v2.0:**
- ✅ Удалены 5 неиспользуемых функций парсинга
- ✅ Удалено 275 строк мертвого кода
- ✅ Очищены неиспользуемые импорты

---

### **downloader.py** (130 строк)
Скачиватель конфигов с проверкой версионирования.

**Функции:**
- `download_file()` - скачивает файл с проверкой кэша
- `clean_config_content()` - очищает контент конфигов
- `download_all_tasks()` - скачивает все файлы из конфига
- `_create_session_with_retries()` - NEW создает сессию с retry

**Улучшения v2.0:**
- ✅ Добавлен retry механизм (3 попытки, экспоненциальная задержка)
- ✅ Улучшена обработка сбоев сети

---

### **ui.py** (280 строк)
Модуль для красивого вывода в консоль.

**Функции:**
- `print_banner()` - выводит красивый баннер
- `print_header()`, `print_subheader()` - выводит заголовки
- `print_success()`, `print_error()`, `print_warning()`, `print_info()` - цветной вывод
- `print_loading()` - анимация загрузки
- `print_results_table()` - таблица результатов
- `print_summary()` - итоговая сводка
- Класс `Colors` - ANSI цветовые коды

**Улучшения v2.0:**
- ✅ Удалена неиспользуемая функция `type_writer()`

---

### **config.py** (75 строк)
Глобальные конфигурационные переменные.

**Переменные:**
- `XRAY_BIN` - путь к Xray бинарнику
- `RESULTS_DIR` - директория результатов
- `TASKS` - список задач тестирования (VPN, Bypass, MTProto)
- `CHROME_UA` - User-Agent

---

### **file_utils.py** (450+ строк)
Утилиты для работы с файлами и конфигами.

**Функции:**
- `has_insecure_setting()` - проверяет наличие слабых параметров
- `split_file_into_chunks()` - разбивает файл на части
- `extract_host_port()` - извлекает хост и порт
- `extract_ip_from_config()` - извлекает IP адрес
- `filter_secure_configs()` - фильтрует ненадежные конфиги

**Статус v2.0:**
- Требует рефакторинга функции `has_insecure_setting()` (250+ строк)

---

### **logger.py** (20 строк)
Простой логгер для вывода информации.

**Функции:**
- `log()` - выводит лог с временной меткой

---

### **constants.py** и **settings.py**
Пусты или содержат минимум данных.
